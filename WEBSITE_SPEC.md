## 1. 프로젝트 개요

### 1.1 사이트 목적

메이크그로스는 이커머스 1인 셀러를 1인 기업가로 육성하는 AX(AI Transformation) 파트너입니다. 이 웹사이트는 잠재 고객(이커머스 셀러)에게 메이크그로스의 서비스를 소개하고, 세미나 신청 및 견적 문의를 받는 것이 핵심 목적입니다.

### 1.2 핵심 메시지

* **메인 카피** : "할 줄 모르는 일은 전부 비용입니다. 못하던 일을 할 수 있게 만드는 파트너."
* **서브 카피** : "외주가 끝나면 시스템도 멈춥니다. 스스로 굴러가는 구조를 만들어 드립니다."
* **포지셔닝** : 이커머스 사업자의 시간과 비용을 되찾아주는 AI 파트너 — 진단 → 구축 → 교육

### 1.3 타겟 유저

* **Primary** : 쿠팡/스마트스토어 1인 셀러 (월 매출 500만~5,000만원)
* **Secondary** : 이커머스 창업 준비자, 소규모 셀러 팀 (2~5인)
* **특성** : 비개발자, 외주 의존 높음, 시간 부족, AI 관심은 있으나 실행 방법 모름

### 1.4 기술 스택

* **Framework** : Next.js 14+ (App Router)
* **Styling** : Tailwind CSS
* **Language** : TypeScript
* **Deployment** : Vercel
* **Form** : 자체 구현 (Google Sheets API 또는 Notion API 연동)
* **Analytics** : Google Analytics 4
* **Font** : Pretendard (한글), Inter (영문)
* **아이콘** : Lucide React

* 배경: 다크 그라데이션 (Navy → Black) 또는 미니멀 라이트
* **페인포인트 나열 모션** : 셀러의 실제 고충이 하나씩 나타나며 사라지는 효과

* `상세페이지 외주 30만원, 수정 요청만 3번째...`
* `밤새 키워드 리서치했는데, 결국 뭘 등록해야 할지 모르겠어요`
* `카톡방 돌며 유통처 찾느라 하루가 또 갔어요`
* `SNS 운영? 할 시간이 어디 있어요`
* → 마지막 전환: **"내 상품, 내 카테고리에 맞는 AI. 이제 직접 만드세요."**
* 메인 카피: **"할 줄 모르는 일은 전부 비용입니다"**
* 서브 카피: "이커머스 사업자의 시간과 비용을 되찾아주는 AI 파트너"
* CTA 2개:
  * [세미나 신청하기] → /seminar (Primary, filled)
  * [견적 문의하기] → /contact (Secondary, outlined)

> ✅ 페인포인트 나열 → 해결 제시 (PAS 프레임워크) 적용.
> 프로젝트 PDF(연매출 10억 셀러, 유통의 정석, 이브레인 템플릿)에서 추출한

> "1인 셀러"로 한정하지 않고 "이커머스 사업자"로 범위 확장 — 소규모 팀, 중소기업도 포함.

**섹션 2: 킬러 모듈 — SNS 콘텐츠 자동화** ★

* ★ 표시 (대표 모듈, 운영 중)
* 설명: "트렌드 기반 주제 기획부터 글·이미지 콘텐츠 생성, 멀티 채널 예약 발행까지 전 과정을 자동화"

## 6. 인터랙션 & 애니메이션

### 6.1 필수 인터랙션

| 요소                   | 애니메이션                                                | 트리거                               |
| ---------------------- | --------------------------------------------------------- | ------------------------------------ |
| 히어로 페인포인트 전환 | 셀러 고충 키워드 페이드인 → "이제, AI가 대신합니다" 전환 | 페이지 로드 (자동 반복)              |
| 통계 카운터            | 0 → 목표 숫자 카운트업                                   | Viewport 진입 (IntersectionObserver) |
| 섹션 fade-in           | opacity 0→1 + translateY(20→0)                          | Viewport 진입                        |
| FAQ 아코디언           | height 0 → auto, 회전 화살표                             | 클릭                                 |
| 후기 캐러셀            | 자동 스크롤 + 수동 넘기기                                 | 자동 (5초) + 터치/드래그             |
| CTA 버튼               | scale(1.02) + 그림자 강화                                 | 호버                                 |
| 네비게이션             | 배경색 변화 (투명→불투명)                                | 스크롤 다운                          |

### 6.2 성능 원칙

* 애니메이션은 `transform`과 `opacity`만 사용 (리페인트 최소화)
* 모바일에서 자동 캐러셀 터치 시 자동 스크롤 일시 정지
* 이미지는 Next.js `<Image>` 컴포넌트 + WebP + lazy loading

## 12. Claude Code 빌드 지시사항

### Superpowers 플러그인 활용 (권장 워크플로우)

Superpowers는 Claude Code 플러그인으로, 코드를 바로 작성하지 않고 **브레인스토밍 → 설계 → 계획 → 실행** 단계를 거치게 해줍니다. `visual-companion` 기능으로 브라우저에서 목업/와이어프레임을 시각적으로 보면서 디자인을 확정할 수 있습니다.

**설치**

```bash
# Claude Code에서 플러그인 설치
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace
```

**추천 빌드 플로우**

```
Step 1: 브레인스토밍 (디자인 확정)
/superpowers:brainstorm
→ 이 기획문서(WEBSITE_SPEC.md)를 입력으로 제공
→ visual-companion 활성화 시 브라우저에서 목업 확인 가능
→ 페이지별 레이아웃, 컬러, 컴포넌트 구조를 시각적으로 검토
→ 승인 후 설계 문서 자동 생성

Step 2: 구현 계획 수립
/superpowers:write-plan
→ 브레인스토밍 결과를 기반으로 파일별 구현 태스크 생성
→ 각 태스크: 파일 경로 + 코드 + 검증 단계 포함

Step 3: 계획 실행
/superpowers:execute-plan
→ 서브에이전트가 태스크를 순차 실행
→ 코드 리뷰 자동 포함

Step 4: 수동 검수 + 배포
→ Vercel로 배포, 모바일/데스크톱 확인
```

**visual-companion 활용 포인트**

* 히어로 섹션 레이아웃 비교 (다크 vs 라이트)
* Before/After 카드 디자인 옵션
* 세미나 랜딩 페이지 섹션 순서 조정
* 모바일 네비게이션 패턴 선택
* CTA 버튼 위치/색상 비교

### 빌드 시 참고 원칙

1. **모바일 퍼스트** : 타겟 유저(셀러)는 모바일 사용 비율이 높음. 반드시 모바일 우선 설계.
2. **로딩 속도** : Lighthouse 90+ 목표. Next.js Image 최적화, 폰트 프리로드, 불필요한 JS 제거.
3. **접근성** : 시맨틱 HTML, alt 텍스트, 키보드 내비게이션, 색 대비 4.5:1 이상.
4. **CTA 가시성** : 모든 주요 섹션 끝에 CTA 버튼. 모바일에서 Sticky Bottom CTA 고려.
5. **데이터 분리** : 콘텐츠 데이터는 `/data/` 폴더에 JSON/TS로 분리하여 비개발자도 수정 가능하게.
6. **placeholder** : `[TODO: 내용]` 형식으로 통일하여 검색·교체 쉽게.
7. **한국어 최적화** : word-break: keep-all, 줄바꿈 자연스럽게, Pretendard 폰트 우선.

## 부록 A: 카피라이팅 톤앤매너 가이드

### 쓰는 말

* "외주비 절감" → 구체적 수치로 ("월 100만원 → 80%+ 절감")
* "자동화" → "시스템이 대신합니다"
* "AI 도입" → "AI를 내 체질로 만드는"
* "교육" → "직접 하실 수 있을 때까지"
* "컨설팅" → "함께 만듭니다"

### 안 쓰는 말

* ❌ "AI Champion" (엔터프라이즈 용어)
* ❌ "조직 문화 변혁" (1인 셀러에게 부적합)
* ❌ "혁신적 솔루션" (추상적)
* ❌ "원스톱 서비스" (식상함)
* ❌ "시너지" (의미 불명확)

## 부록 B: 사업계획서와의 일관성 체크 포인트

웹사이트 콘텐츠는 사업계획서(v2)와 다음 항목이 반드시 일치해야 합니다:
