# DiagnosisTool 퍼널 리디자인 + 홈 섹션 재배치

> **목적**: 셀러 잠재고객의 사용자 여정을 자연스러운 설득 흐름으로 재설계
>
> **스코프**: 홈페이지 섹션 순서 + DiagnosisTool 4스텝 UX + StickyBottomCTA + 듀얼 CTA (무료/유료)
>
> **작성일**: 2026-04-09

---

## 1. 문제 정의

### 현재 홈페이지 흐름
```
Hero → VideoIntro → PainCost → BotDemo → HomePricing → DiagnosisTool → HowItWorks → WhyUs → FAQ
```

### 문제점
1. **신뢰 전에 결제 유도**: HomePricing(가격)이 HowItWorks/WhyUs(신뢰) 앞에 위치
2. **진단기 진입 맥락 부재**: StickyBottomCTA "무료 체험 시작" → 갑자기 설문 화면 → 혼란
3. **Step 0 유도 약함**: "내 하루는 어디에 쓰이고 있을까?" — 왜 해야 하는지, 뭘 받는지 불명확
4. **스텝 간 스토리 단절**: 각 스텝이 기계적 데이터 수집이지 감정적 여정이 아님
5. **단일 CTA**: 결과 화면에서 무료/유료 경로가 분명하지 않음
6. **VideoIntro 빈 슬롯**: 영상 미보유 상태에서 placeholder가 노출됨

---

## 2. 설계

### 2.1 홈페이지 섹션 재배치

**변경 후:**
```
Hero                    — 훅: "이거 내 얘기인데?"
  → PainCost            — 각성: "이렇게 심각했어?"
  → BotDemo             — 발견: "이런 게 있었어?"
  → HowItWorks          — 신뢰: "어떻게 작동하지?"
  → WhyUs               — 확신: "이 사람들 믿을 만하네"
  → DiagnosisTool       — 행동: "내 상황은 어떨까?"
  → HomePricing         — 결정: "이 가격이면 해볼 만하네"
  → HomepageFAQ         — 해소: 마지막 의문 제거
```

**변경 사항:**
- `VideoIntro` 제거 (영상 에셋 미보유, 추후 Hero 통합 또는 별도 복원)
- `HowItWorks` + `WhyUs`를 DiagnosisTool 앞으로 이동
- `HomePricing`을 DiagnosisTool 뒤로 이동
- 심리 순서: 인지 → 공감 → 발견 → 신뢰 → 개인화 → 결정

**파일 변경**: `src/app/page.tsx` — import 순서 + JSX 순서 변경

### 2.2 DiagnosisTool 스텝별 리디자인

#### Step 0 — 인트로 (신규 화면 → 기존 Step 0 대체)

**현재**: 다크 카드에 카테고리/매출 선택 + "분석 시작하기"
**변경**: 보상 프리뷰 + 감정 훅 + 기존 입력 유지

```
[상단 — 감정 훅]
  eyebrow: "무료 · 90초 · 이메일만 있으면 됩니다"
  h2: "대부분의 셀러는\n자기가 어디에 돈을 쓰는지 모릅니다"
  sub: "3분이면 매달 얼마를 AI로 줄일 수 있는지 알 수 있습니다"

[중단 — 보상 프리뷰 3카드]
  ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
  │ 📊 절감 리포트   │ │ 🤖 봇 체험 가이드│ │ 📬 AI 셀러 인사이트│
  │ 내 외주비 중     │ │ 텔레그램 봇      │ │ 주간 뉴스레터     │
  │ AI 전환 가능 금액│ │ 온보딩 자료 제공  │ │ 자동 구독          │
  └─────────────────┘ └─────────────────┘ └─────────────────┘

[하단 — 기존 입력]
  카테고리 select
  매출 규모 select
  CTA: "내 루틴 분석하기 →"
```

**디자인 톤**: 기존 다크 카드(#1A1A1A) 유지. 보상 카드는 `bg-white/[0.06] ring-1 ring-white/10` (다크 위 글래스).

#### Step 1 — 자각 ("하루 중 이 시간이 전부 비용입니다")

**기존 유지 항목**: 슬라이더 7개 + 도넛 차트 + 요약

**추가/변경**:
- 실시간 시급 환산 텍스트: `"소싱에 1.5시간/일 → 연간 547시간"` (hours × 365)
- 스텝 하단 브릿지 카피: `"이 시간에 가격표를 붙여볼까요?"` → 다음 버튼 위에 배치
- 다음 버튼 텍스트: "다음 →" → **"외주비 확인하기 →"**

#### Step 2 — 충격 ("이 돈이 매달 빠져나갑니다")

**기존 유지 항목**: 외주 체크박스 + 비용 선택 + 요약 패널

**추가/변경**:
- h3 서브 카피 중복 수정: 현재 "이 외주비가 매달 나갑니다" 2번 반복 → 서브는 `"현재 외주하고 있는 업무를 체크해주세요"`
- 스텝 하단 브릿지 카피: `"이 중 AI가 대신할 수 있는 금액을 보여드릴게요"`
- 다음 버튼 텍스트: "결과 보기 →" → **"AI 절감액 확인 →"**

#### Step 3 — 보상 (결과 + 듀얼 CTA)

**기존 유지 항목**: 현황 요약 카드 2개 + AI 전환 영역 + 절감 요약 다크 블록

**변경 — 듀얼 CTA 구조**:

현재의 "14일 무료 체험" 단일 CTA + 이메일 수집을 **명확한 2갈래**로 분리:

```
[절감 요약 다크 블록] (기존 유지)

[듀얼 CTA 영역 — 신규]
┌───────────────────────────────────┐  ┌───────────────────────────────────┐
│  무료로 시작하기                    │  │  지금 바로 도입하기                 │
│                                   │  │                                   │
│  ✓ 절감 리포트 이메일 발송         │  │  ✓ AI 셀러 비서 봇 즉시 세팅       │
│  ✓ 텔레그램 봇 체험 가이드         │  │  ✓ 14일 무료 체험 후 월 49,000원   │
│  ✓ 주간 AI 셀러 인사이트           │  │  ✓ 1:1 온보딩 지원                 │
│                                   │  │                                   │
│  [이메일 입력]                     │  │  [결제하기 — 토스페이먼츠]          │
│  [개인정보 동의]                   │  │                                   │
│  [무료 리포트 받기 →]              │  │  [월 49,000원으로 시작 →]          │
│                                   │  │                                   │
│  border-[#E0E0E0] bg-[#F8F8F8]    │  │  border-[#1A1A1A] bg-[#1A1A1A]    │
│  (라이트 카드)                     │  │  (다크 카드, 강조)                  │
└───────────────────────────────────┘  └───────────────────────────────────┘
                        모바일: 세로 스택 (무료 먼저, 유료 아래)
```

**제거**: 기존 Step 3의 중복 CTA들 (무료 체험 CTA, 이메일 섹션, 웨비나/서비스 버튼이 산재) → 듀얼 CTA 하나로 통합.

**유료 결제**: 토스페이먼츠 PG 연동은 이번 스코프에서 **UI만 구현** (버튼 + 가격 표시). 실제 결제 연동은 별도 스펙. 클릭 시 `/contact`로 이동하는 중간 단계.

### 2.3 StickyBottomCTA 리워딩

**현재**: `"AI 직원, 14일 무료 체험 → 월 49,000원"` + `"무료 체험 시작"`
**변경**: `"내 셀러 루틴, AI로 얼마나 줄일 수 있을까?"` + `"무료 진단하기"`

- href: `/diagnosis` (유지)
- 기대와 결과 일치: "무료 진단" → 실제 진단 화면

### 2.4 기존 이메일 수집 필드 정리

현재 Step 3에 있는 선택 필드들 (고민, 셀러 활동 현황, AI 관심도, 하고 싶은 말):
- **무료 경로**: 이메일 + 개인정보 동의만 (마찰 최소화)
- **선택 필드 3개**(painPoint, sellerStatus, aiInterest): 이메일 제출 후 "추가 정보" 옵션으로 이동 (선택적, 제출 성공 화면에서)
- **freeComment**: 제거 (초기 리드 수집에서 자유 텍스트는 마찰)

## 4. 변경하지 않는 것

- DiagnosisTool의 계산 로직 (`computeResults`, 슬라이더 상태 관리)
- 도넛 차트 컴포넌트
- Google Sheets / N8N 제출 로직
- 기존 디자인 시스템 (모노크롬 + 딥네이비, Supanova easing)
- 다른 페이지 (/services, /seminar, /about 등)

## 5. 성공 기준

- 셀러가 홈페이지 스크롤 시 자연스럽게 "인지 → 공감 → 신뢰 → 행동" 순서를 경험
- DiagnosisTool Step 0에서 "왜 해야 하는지 + 뭘 받는지"가 3초 안에 파악 가능
- Step 3에서 무료/유료 경로가 명확히 분리되어 혼란 없음
- 기존 Supanova 모션 퀄리티 유지 (easing, stagger, blur transition)
